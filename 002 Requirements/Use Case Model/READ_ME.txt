For the Use Cases the following naming convention is used:

UCx_y_z_ServiceName+Use+Case+Specification, where

x - Each e-TrustEx module has a number. (e.g. the toolbox services have "1", e-PRIOR "2")
y - Each e-TrustEx modules has a set of business services/ document type supported. 
     This is the number of a service within a module. e.g. Send Document Service has number1)
z - For each document type there are two specific Use Cases. 
     1 - Use Case that includes the specific business processing.
     2 - Use Case that includes the specific validation ( xsd, schematron, parent document check)
