2. Implementation of context/value association to Schematron - Crane-Constraints2Schematron.xsl

Path: Crane-Constraints2Schematron.xsl

$Id: Crane-Constraints2Schematron.xsl,v 1.8 2008/11/11 16:32:40 gkholman Exp $

A Schematron-based implementation of value validation supporting the OASIS Code List Representation Technical Committee specification of context/value association files. This implemenation assumes an unspecified query binding or a query Binding of "xslt" as representing any valid XPath 1.0 pattern defined by XSLT 1.0 production [1].

This stylesheet calls out to external template rules to accommodate the naming of metadata and the format of external code list enumerations, thus it is not hardwired to any particular document vocabulary or to the genericode representation of code lists.

It should not be necessary to modify this file in order to support any definition of business document meta data or value list vocabulary.

Implementation note: literal result elements are not used in this fragment because of the desire to emit Schematron elements in the default namespace while needing the namespace declared for matching purposes.

Implementation restriction: at this time this script is checking individual code list association files for having non-conflicting prefixes for namespaces, but it is not checking the entire suite of prefixes for namespaces for every included file for unique usage of prefix values in the generated result. This may go unreported and cause a problem in the Schematron processing.

3. CVA support for UBL instance-level meta data - Crane-UBL-Metadata.xsl

Path: Crane-UBL-Metadata.xsl

$Id: Crane-UBL-Metadata.xsl,v 1.4 2008/11/11 16:33:28 gkholman Exp $

Support for an implementation of the OASIS Code List Value Validation Methodology.

This fragment determines the names of UBL meta data information items related to code list information items.

ccs:document-model-metadata (xsl:template)

For a given information item, engage the checking of various metadata properties supported by the external code list implementation by calling check-list-property for every possible instance-level meta data value.

  • for an attribute matching @currencyID:

    • @currencyCodeListVersionID mapped to genericode <Version> using the property Version

  • for an attribute matching MeasureType/@unitCode:

    • @unitCodeListVersionID mapped to genericode <Version> using the property Version

  • for an attribute matching QuantityType/@unitCode:

    • @unitCodeListID mapped to genericode <Version> using the property ID

    • @unitCodeListAgencyID mapped to genericode <Agency><Identifier> using the property AgencyID

    • @unitCodeListAgencyName mapped to genericode <Agency><LongName> using the property AgencyLongName

  • for an element named <xxxxxCode> based on a code type:

    • @listName mapped to genericode <LongName> (first occurrence) using the property LongName

    • @listID mapped to genericode <LongName @Identifier='listID'> or just <LongName> (first occurrence) using the property listID

    • @listVersionID mapped to genericode <Version> using the property Version

    • @listSchemeURI mapped to genericode <CanonicalVersionUri> using the property CanonicalVersionUri

    • @listURI mapped to genericode <LocationUri> using the property LocationUri

    • @listAgencyName mapped to genericode <Agency><LongName> using the property AgencyLongName

    • @listAgencyID mapped to genericode <Agency><Identifier> using the property AgencyID

  • for an element named <yyyyyID> based on an identifier type:

    • @schemeName mapped to genericode <LongName> (first occurrence) using the property LongName

    • @schemeVersionID mapped to genericode <Version> using the property Version

    • @schemeURI mapped to genericode <CanonicalVersionUri> using the property CanonicalVersionUri

    • @schemeDataURI mapped to genericode <LocationUri> using the property LocationUri

    • @schemeAgencyName mapped to genericode <Agency><LongName>AgencyLongName

    • @schemeAgencyID mapped to genericode <Agency><Identifier>AgencyID

  • for all other attributes and elements defined by UBL

    • it is assumed there is no meta data that describes the value set from which an information item's values are constrained that is suitable for checking against the meta data found in value set

Parameter item

(deprecated) The normalized @item attribute from the association (for namespace-safe reasons, $item should not be used).

Parameter list

<ValueList> node from the association.

Parameter attr

true/false that the item being tested is an attribute.

Parameter name

the local name of the addressed item (without a prefix).

Parameter ns

the namespace of the addressed item (not the prefix).

This fragment supports external code list expressions of code list values written using either genericode 0.4 or genericode 1.0, where those files use only simple values for keys.
